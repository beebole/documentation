# Documentation Audiences

Use this context to tailor language, depth, and assumptions based on who the page is for.

## Primary audience: Account administrators

The main readers of the Documentation tab. These are office managers, team leads, project managers, accountants, HR staff, or business owners who set up and configure Beebole for their organization.

**What they need:** Full picture of account setup, configuration, permissions, billing, and all features.

## Secondary audiences

| Audience | Typical content | Tab |
|----------|----------------|-----|
| **Team leaders / Project managers** | Approving timesheets, running reports, managing their team | Guides |
| **Employees** | Recording time, submitting timesheets, requesting time off | Guides |
| **Developers** | API authentication, queries, mutations | API |

## Writing for each audience

- **Admin pages** — Can reference settings, permissions, and configuration. Admins need the full picture.
- **Manager pages** — Focus on day-to-day oversight tasks. Assume the account is already set up.
- **Employee pages** — Simplest language. Assume minimal familiarity with Beebole. Focus on the specific task.
- **API pages** — Technical language is fine. Assume developer knowledge.

If a page serves multiple audiences, lead with the most common use case. Use callouts or separate sections for role-specific details.
