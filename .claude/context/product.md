# Beebole — Product Overview

Use this context to understand what Beebole does and how its concepts relate to each other. App URLs, support email, and source code repo details are in CLAUDE.md.

## Core capabilities

- **Timesheets** — Weekly or monthly time entry for employees, with approval workflows
- **Projects & Tasks** — Organize work by company, project, sub-project, and task
- **Absences** — Leave management with types, quotas, accruals, and public holidays
- **Reporting** — Built-in reports with filters, grouping, and export
- **Billing & Cost** — Track billable/non-billable time, set rates, and calculate costs
- **Permissions** — Role-based access control (Admin, Manager, Employee) with granular settings
- **Integrations** — Connects with third-party tools (see Integrations tab)
- **API** — GraphQL API for developers

## Key concepts

- **Company** — The top-level organizational unit. Projects belong to companies.
- **Project** — A container for time tracking. Can have sub-projects and tasks.
- **Task** — An independent planning entity (not a sub-element of projects). Tasks can be assigned to one or more projects.
- **Group** — A way to organize people (e.g., by department or team).
- **Work schedule** — Defines expected working hours per day/week for users.
- **Approval workflow** — Managers review and approve/reject submitted timesheets.
