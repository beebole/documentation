# Feedback inbox

When you review a documentation page and have feedback to share, drop your marked-up file into this folder: `docs/feedback/`.

## How to submit feedback

1. Copy the page you reviewed (or write fresh prose notes — any format works).
2. Mark it up however you want: inline strikethroughs, additions, margin notes, bullet lists referencing page paths.
3. Save it in `docs/feedback/` in this repo. Name the file however you like — no convention required.
4. Commit and push like you normally would.

That's it. No tooling, no format, no ticket. Just a markdown file in this folder.

## What happens next

Yves periodically triages this folder and files each note into the right place in the documentation conventions. Once processed, your file is removed from this folder — git history keeps the original around if anyone needs to look back.

## Scope

Feedback on English pages and feedback on French/Spanish translations both go in this folder. Single typos are just as welcome as broader structural feedback.
