## Introduction

This migration guide covers everything you need to know to move your account to the new Beebole platform. You'll learn how the new and legacy systems coexist during the transition, what's new and different, and how the updated API and architecture might affect your planning.

<Info>
  Your legacy Beebole account remains fully active throughout this process — no disruption to your current usage or data.
</Info>

## What the new platform means for you

There is no rush to switch to the new system, though you can start a free trial at any time while your legacy account continues to run. Here are the main things you should know:

- Your legacy data stays safe and untouched until you're ready to migrate.
- Migration will be done manually by our team, one account at a time. Contact us when you're ready to switch, and we'll get back to you as soon as possible.
- Users can log in to both systems using the same page, even with multiple accounts.
- We recommend taking advantage of this opportunity to reconsider the architecture of your projects, tasks, and teams. Migration is a chance to make a change.
- If you are using Beebole's API, your integrations and custom workflows will need to be updated. We're here to help, but please bear with us. In the meantime, everything will continue working in your current account.

<Info>
  The migration itself is handled by Beebole support. Historical projects, people, and time records are migrated manually on a case-by-case basis.
</Info>

## Create a trial account and explore the new platform

Customers who want to try the new system can create a free trial account without affecting the legacy system.

- Sign up for a free trial at [https://app.beebole.com/signup](https://app.beebole.com/signup). It's fine to use the same email address as your existing account.
- Use the trial account to explore new features, create time entries, discover resource planning, and play around with reports. Don't worry; when you're ready to permanently switch, you can start fresh with another clean account.
- As with any new system, we anticipate some initial technical kinks. Please share them with us, along with any other feedback you have!

## Logging in when both systems exist

The new and legacy systems share the same [login page](https://app.beebole.com/signin). Beebole will detect which system an account belongs to.

- If your email is only registered in the legacy system, you go to the legacy account.
- If your email is only registered in the new platform, you go to the new account.
- If you have both accounts, you get to choose which system to access.

## Rethink your architecture for the new platform

The new platform is a good moment to revisit how your organization uses, or could use, projects, tasks, teams, tags, and custom fields. Ask yourself:

- Does your current project hierarchy still reflect how work is delivered?
- Are people efficiently grouped for reporting and approvals?
- Have your timesheet settings, localization, costs, and billing rates become convoluted and confusing?
- Are there custom fields you need on projects, people, tasks, or time entries to improve data accuracy or invoicing?

Our [Key Concepts](/help/documentation/concepts) page is a great resource for understanding Beebole's core entities (projects, people, tasks, and tags) and how they relate to each other. Try mapping out your structures and settings ahead of time and, when in doubt, reach out to us.

## New API and developer readiness

The new platform includes a [GraphQL API](/help/api/introduction) for querying and mutating Beebole data. This is the primary developer-facing change for customers with integrations or automation.

Review `/help/api/introduction` for the API basics and authentication model. Then, start planning the migration of any legacy integrations as part of the manual account migration. 

<Info>
  The API is a major reason to evaluate the new platform early. It is also the right time to document your current integration points and decide which data should move first.
</Info>

## What's Changed in the New System

Below is an overview of key features and differences between the legacy and new Beebole systems.

### Time Tracking

| Topic | What's new |
| :-- | :-- |
| **Timesheet submission** | Users now submit a full timesheet period (e.g., a whole week), not individual time entries. Submitting an empty or incomplete timesheet locks that entire period — no entries can be added afterwards. |
| **Timesheet periods** | Fixed daily/weekly/monthly views are gone. Admin configure a flexible period (weekly, bi-weekly, 1st–15th, etc.) that matches your company's actual policy. |
| **Time entry format** | Hours and decimals still work, but you can now also log time in days or as a percentage of a day (e.g., "I spent 20% of today on this"). |
| **Work from home flag** | A dedicated WFH flag is now available directly on each time entry. |
| **Non-billable flag** | Employees can now mark individual entries as non-billable. |
| **Auto-submit** | Admins can configure a timesheet to be submitted automatically after a set number of days, useful for locking periods when employees forget. |
| **Admin direct edits** | Admins can now edit submitted or approved time entries directly without rejecting them first. All changes are logged. |
| **Custom fields on time entries** | Add custom fields to time entries to capture additional data (e.g., units produced, items completed) for use in reports. |
| **Timesheet health score** | Replaces the old "on-time submit" metric. Calculates a score per employee based on late submissions, missed sheets, and rejections — displayed as a gauge visible only to managers. |

### Absences & Time Off

| Topic | What's new |
| :-- | :-- |
| **Accrual policies** | **Coming soon!** The system can automatically add holiday quota based on time worked (e.g., earn X days per month worked). |
| **Carry forward rules** | Configure whether unused holiday days can roll over to the next year, and for how long. |
| **Absence unit** | You can now configure each absence type to be requested in either days/half-days or hours, not one or the other across the board. |
| **Public holidays** | No more manual entry each year. The system automatically manages public holidays by country and region. You can still modify or add custom holidays on top. |

### Approvals

| Topic | What's new |
| :-- | :-- |
| **Multi-stage approval workflow** | Fully configurable now. Define as many stages as needed (e.g., project manager → team leader → admin), separately for working time and absences. |
| **Quorum rules** | At each stage, you can set whether _all_ approvers or only _one_ must approve before advancing. |
| **Approval story** | Every timesheet now has a full log of all actions taken — approvals, rejections, comments, change requests — so you can always see exactly where things stand. |
| **Blocked stage visibility** | You can see precisely which stage an approval is stuck at and who hasn't acted yet. |
| **Email action buttons** | Approval emails now include direct action buttons — click to approve without opening the app. |
| **Approval reminders for managers** | Managers now receive periodic reminders for approvals that have been pending. |
| **Notification digests** | Instead of individual notifications for every event, you can opt for a periodic digest (daily, weekly) delivered via email, Slack, or in-app. |
| **Mobile approvals** | The mobile app now includes timesheet approval. |

### Planning & Tasks

| Topic | What's new |
| :-- | :-- |
| **Kanban board** | Tasks can be managed in a Kanban view with configurable statuses. |
| **Gantt chart/resource planning** | Assign tasks to people with a % allocation (e.g., 50% FTE), see utilization per person, and identify over/under-allocation. |
| **Task dependencies** | Tasks can be linked so one can't start before another finishes. |
| **Recurring tasks** | Set up tasks that repeat on a schedule (e.g., every Friday). |
| **Timesheet integration** | Tasks assigned to you automatically appear in your timesheet. If planning is kept up to date, employees may just need to confirm and submit. |
| **Custom fields on tasks** | Add extra structured information to tasks beyond the description. |
| **Task ownership** | Each task has one owner. Unassigned tasks are visible to defined potential owners, and the first person to log time against one becomes the owner. |

### Expenses

| Topic | What's new |
| :-- | :-- |
| **Project-level expenses** | Expenses can be trakced on a project and factored into budget tracking. |
| **Expense types** | Define expense types with currency or non-currency units (e.g., miles for travel). |
| **Markup settings** | Set a billing markup on expenses (e.g., charge customers 120% of cost). |
| **Budget impact toggle** | Choose whether an expense type counts toward the project budget or not. |

### People & Roles

| Topic | What's new |
| :-- | :-- |
| **Unlimited custom roles** | The five fixed roles are gone. You can use default roles, or create as many custom ones as you need. |
| **Bulk operations** | Bulk-invite, bulk-archive, and bulk-unarchive employees. |
| **Admin takeover** | Admins can now impersonate any employee to troubleshoot issues and see exactly what they see. |
| **Pre-onboarding setup** | You can create and configure an employee before they start. They'll stay in "archived" status until they're activated. |

### Projects & Structure

| Topic | What's new |
| :-- | :-- |
| **Unlimited project levels** | The fixed 3-level structure (customer → project → subproject) is gone. You can have as many levels as needed. |
| **Multiple parallel columns** | In addition to unlimited depth, you can now add multiple independent project columns to the timesheet (e.g., Customer \+ Task \+ Location). |
| **Project availability modes** | Choose either: _visible to all by default_ (with the option to exclude specific groups), or _hidden by default_ (must be explicitly assigned). Works for projects, absences, and expenses independently. |

### Tags & Configuration

| Topic | What's new |
| :-- | :-- |
| **Tags as configuration groups** | Tags can carry configuration (e.g., which activities a team sees, which holidays apply), and a person, project, or entity tagged with multiple tags receives a _combination_ of all their configurations. |
| **Multiple group membership** | In the legacy system, entities could only belong to one configuration group. Now, a person in both "UK" and "Marketing" gets UK bank holidays _and_ Marketing-specific activities simultaneously. |
| **Unlimited tags and levels** | Tag hierarchies are unlimited in depth and number (e.g., Organization → Region → Team). |
| **Org structure via tags** | You can model your entire org chart using a tag hierarchy, making it easier to manage than assigning managers person by person. |

### Billing & Cost

| Topic | What's new |
| :-- | :-- |
| **Daily rates** | Billing and cost rates can now be defined per day (not just per hour). |
| **Fixed recurring fees** | Define a fixed cost or billing rate (e.g., a salary), which is applied for the defined period, regardless of hours worked. |
| **Non-billable / zero-cost activities** | You can now flag specific activities as non-billable or as having no cost. |
| **Budgets** | Budgets can now be set for costs, billing, or hours and split by person or project. |

### Reporting

| Topic | What's new |
| :-- | :-- |
| **Saved reports with categories** | Reports can now be created, saved, and organized into categories. A category can share a default date range and data filter that applies to all reports within it. |
| **Table \+ chart toggle** | Each report can display results as a data table, a chart (bar, pie, etc.), or both simultaneously. |
| **Spreadsheet add-on (new approach)** | Report configuration is done in the interface and exported as a URL/config to the Excel/Sheets add-on. The add-on always fetches fresh data based on that config — no more maintaining column mappings inside the spreadsheet. |
| **Budget status reports** | Easy check the budget status and planned budget on any project for costs, billing, or time in the reports. |

### Communication & Notifications

| Topic | What's new |
| :-- | :-- |
| **Journal (activity feed)** | Every entity (person, project, task, etc.) has a journal — a threaded activity feed where you can comment, attach files, mention people, and track changes. |
| **Email reply-to-comment** | Reply directly to a journal notification email to add a comment — no need to open the app. |
| **Customizable email templates** | All system emails (invitations, reminders, approval requests) can now have customized content, including dynamic placeholders (e.g., employee name). |
| **Web push notifications** | Browser push notifications in addition to email. |

### Other / Platform

| Topic | What's new |
| :-- | :-- |
| **Integrations** | Native integrations now include Asana, Jira, Monday.com, Linear, QuickBooks, BambooHR, plus webhooks, with more to come! |
| **Passkeys & passwordless login** | Login via passkey or a PIN-code email link — no passwords required. |
| **Organization branding** | Upload a logo and choose an accent colour. Your sign-in page will reflect your brand. |
| **Undo/redo (global)** | A global undo/redo that works across the entire app — including navigation back to where the change was made. |
| **Attribute copy/paste** | Duplicate a project (or other entity) with all its configuration intact, as a starting template. |
| **Bullk import with copy/paste** | Add multiple people, projects, etc. at once by copy and pasting spreadsheet rows. |
