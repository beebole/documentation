Beebole's project structure is the backbone of your time tracking data. Whether you are tracking billable client work or internal administrative activities, everything starts with a project. Projects are what your team logs time against — and every report, budget, and billing calculation flows from this structure.

<Info>
  Projects in Beebole can represent anything your team works on: client engagements, internal initiatives, recurring department tasks, or overhead activities.
</Info>

---

## Understanding the project hierarchy

Beebole uses a flexible hierarchy with no depth limit. You can organize your work into:

- **Categories** — Top-level groupings. A new account starts with **Customer**, **Internal**, and **Activity**.
- **Projects** — Specific engagements or work streams within a category (e.g., **Acme Corp**, **Website Redesign**, **Training**).
- **Subprojects** — Smaller pieces of work within a project. You can nest these as deep as you need.

<Tip>
  Before creating your structure, map your hierarchy on paper — especially if you have multi-level subprojects or complex billing requirements. While easy to create, restructuring later can be time-consuming.
</Tip>

## Creating projects

<Steps>
  <Step title="Open the Projects page">
    Click **Projects** in the left sidebar.
  </Step>
  <Step title="Pick or create a category">
    Every project belongs to a category. Open the category you want the project in — or create a new category first if you need a new grouping. Adding a project always places it in the category that's currently open; there is no separate step to assign a category afterwards.
  </Step>
  <Step title="Add the project">
    Click the **+** button at the top of the list, enter a name, and click **Add**. The project is now available for time tracking. To create a subproject, open the parent project and add the new project from there.
  </Step>
  <Step title="Bulk import (optional)">
    To add multiple projects at once, use the **Import** option. You can copy and paste data directly from Excel or Google Sheets.
  </Step>
</Steps>

## Configuring project settings

Once a project is created, click its name to access detailed settings:

- **Manager** — Assign people to oversee the project and approve time entries.
- **Members** — Control which team members can log time to this project.
- **Tags** — Categorize the project for reporting and organizational grouping. See [Tags](/help/documentation/tags).
- **Custom fields** — Add project-specific metadata (e.g., "Priority", "Region"). See [Custom fields](/help/documentation/custom-fields).

## Billing rates per project

Each project can carry its own billing rates, independent of any rates defined at the organization or tag level. Open the project, go to the **Billing** attribute panel, and click **Add** to create a new rate. You can choose between a fixed fee, an hourly rate, or a daily rate. Use **Split by persons** to define different rates for individual team members working on the project, or **Split by tags** to apply rates based on tag membership.

A rate set directly on a project takes priority over rates inherited from tags or the organization. This means you can leave defaults at the organization level and only override them for projects with special commercial arrangements. See [Billing rates](/help/documentation/billing) for the full priority order and repeat/date-range options.

---

## Cost rates per project

Project-level cost rates let you track the internal cost of delivering work on a specific project, separately from your organization-wide defaults. Open the project and go to the **Cost** attribute panel. The setup mirrors billing: choose between hourly, daily, or fixed-fee methods, and optionally split by person or tag to reflect varying internal costs across your team.

Cost data feeds directly into budget tracking and profitability reports. If a project has no cost rate defined, Beebole falls back to any rate set on the person or at the organization level. See [Cost rates](/help/documentation/costs) for details on the inheritance chain.

## Expense types per project

By default, all expense categories defined in your organization are available for recording expenses. You can restrict or expand this on a per-project basis using the **Show or hide** attribute on the project.

<Steps>
  <Step title="Open the project settings">
    Click the project name to open its attribute panel.
  </Step>
  <Step title="Go to Show or hide">
    Find the **Show or hide** section. On a project it lists expense types and custom fields. (Absence types and tasks appear in **Show or hide** on people, not on projects.)
  </Step>
  <Step title="Toggle expense types">
    Enable or disable individual expense categories for this project. Disabling an expense type prevents team members from selecting it when recording expenses against this project.
  </Step>
</Steps>

You can also assign a project to an expense type directly from the expense type's own settings — use the **Potential owners** attribute on the expense type. See [Expenses](/help/documentation/expenses) for full details.

## Custom fields on projects

Custom fields let you attach structured metadata to a project — for example, a client contract number, a region, a priority level, or any other data point your organization needs for reporting.

Open the project and go to the **Custom fields** attribute panel. The custom fields available for projects appear there automatically — you don't add fields from the project itself. Simply fill in the value (text, number, date, etc.) for each field. Which fields appear is controlled by the field definitions: create fields and set their entity visibility in **Settings** > **Custom Fields**.

Custom field values on a project appear in reports and exports, making it easy to filter, group, or segment your data by any dimension you define. See [Custom fields](/help/documentation/custom-fields) to learn how to create and manage field definitions.

## Timesheet settings per project <Badge>Admin only</Badge>

Every project has a **Timesheet settings** attribute that lets you override the organization-wide time tracking defaults for that specific project. This is useful when different projects require different submission cadences, time entry rules, or reminder schedules.

The settings are grouped into four tabs:

- **Period & submission** — Override the timesheet period (weekly, bi-weekly, monthly, etc.), set an auto-submit delay with **Auto-submit timesheets after (days)**, and control submission rules such as **Allow partial timesheet submission** or **Limit future time entries to time off requests**.
- **Time entry** — Change the **Unit for time entry** (hours or days), the **Duration format**, and the **Minimum time interval accepted**. Enable or require the timer (**Enable timer for time tracking** / **Require timer for time entries**), start/end time capture (**Record start and end times** / **Require start and end times**), and **Require comments on time entries for submission**.
- **Record time on…** — Restrict which project categories or task categories team members can log time against when working on this project.
- **Reminders** — Configure a **Timesheet reminder** (sent at the start of the next period or end of the current one) and an **Approval reminder** sent to approvers a set number of days after the initial notification.

Each setting shows an inheritance indicator: if a value is inherited from the organization, a tag, or a parent project, you can see the source and reset it to the inherited default at any time.

<Info>
  Settings defined at the project level override the organization defaults only for team members working on that project. Other projects are unaffected.
</Info>

## Moving a project to a different category

You can reorganize your structure at any time from the project's action menu.

<Steps>
  <Step title="Open the Projects page">
    Click **Projects** in the left sidebar.
  </Step>
  <Step title="Open the action menu">
    Right-click the project name — or open the **⋯** action menu that appears on hover.
  </Step>
  <Step title="Change the parent">
    Select **Change parent** and choose the new category or parent project. You can also use **Cut** on the project and **Paste** it under the new parent. The change takes effect immediately.
  </Step>
</Steps>

<Warning>
  Moving a project to a different category may affect reports that are filtered by the original category. Review any saved reports or dashboards after reorganizing your hierarchy.
</Warning>

---

## Archiving and deleting projects

<Warning>
  **Deleting** a project permanently removes all associated time entries. Use **Archive** instead to retire completed projects while preserving historical data for reports.
</Warning>

To archive a project, right-click the project name — or open its **⋯** action menu — and select **Archive**. Archived projects no longer appear in timesheets or active lists, but their data remains available in reports.

You can unarchive a project at any time to make it active again.

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Is there a limit to how many levels I can nest?">
    No. Beebole supports unlimited nesting depth. However, for the best user experience, we recommend keeping it under four levels.
  </Accordion>
  <Accordion title="Can I restrict which employees see certain projects?">
    Yes. You can control project visibility through [Assignments](/help/documentation/assignments) and [Roles & permissions](/help/documentation/roles-authorisations).
  </Accordion>
  <Accordion title="Can I set different billing rates for different people on the same project?">
    Yes. Use the **Split by persons** option in the project's billing settings. See [Billing rates](/help/documentation/billing) for details.
  </Accordion>
  <Accordion title="What happens to timesheet settings when I move a project?">
    Timesheet settings you have set directly on the project travel with it. Settings that were inherited from the original category will now be inherited from the new parent instead.
  </Accordion>
  <Accordion title="Can I copy billing or cost rates from one project to another?">
    Yes. In the billing or cost attribute panel, use the **Copy** action to duplicate rates from another project, person, or tag. This is useful when several projects share the same rate structure.
  </Accordion>
</AccordionGroup>
