## Step 1: Create Your First Project

**Objective:** Enable managers to define the work structures they need to track, ensuring data is organized for future reporting. 

**Writing Guidance:** Define "Project" broadly (client work, internal tasks, etc.). Emphasize that projects are the primary bucket for time entries. 

**Internal Linking:** Projects, Timesheets. 

**Visual Asset Suggestion:** Screenshots of the 'New Project' modal with fields to be filled highlighted. Screenshots of configurations after project creation.

### Component Implementation

<Steps>
  <Step title="Navigate to Projects">
    Select the **Projects** tab from the main sidebar and click the **\+ New Project** button.
  </Step>
  <Step title="Define Project Details">
    Enter a project name and assign it to a client. You can also set a budget or start date at this stage.

    ![Creating a new project in Beebole](/images/create-project-modal.png)
  </Step>
  <Step title="Save and Activate">
    Click **Save**. The project is now available for time tracking.
  </Step>
</Steps>

## Step 2: Organize with Tags

**Objective:** Help users implement a secondary layer of data categorization (e.g., Departments, Delivery Models, Service Types) for granular reporting. 

**Writing Guidance:** Explain the difference between Tags and Custom fields. Give some guidance about which things should or should not be Tags. Use "Departments" as the primary example. 

**Internal Linking:** Tags, Custom Fields, Work Schedules, Billing, Costs, Time Off, Timesheet Settings

**Visual Asset Suggestion:** Screenshot of the Tag management screen showing a list of categorized tags.

### Component Implementation

<Info>
  Tags allow you to filter reports by specific criteria across multiple projects, such as "Research" or "Meeting."
</Info>

<Steps>
  <Step title="Access Tag Settings">
    Go to **Settings \> Tags**.
  </Step>
  <Step title="Create a Tag Group">
    Create a group (e.g., "Work Type") and add individual tags within it.
  </Step>
</Steps>

---

## Step 3: Add Your Team

**Objective:** Populate the workspace with users so that time tracking can begin. 

**Writing Guidance:** Focus on the invitation process. Mention that roles and other settings can be adjusted later to control permissions. 

**Internal Linking:** People, Time Off, Billing, Costs, Work Schedules, Tags, Roles and Authorizations. 

**Visual Asset Suggestion:** Screenshot of the 'Add Person' screen focusing on the 'Role' dropdown.

### Component Implementation

<Steps>
  <Step title="Open People Module">
    Click on **People** in the navigation menu.
  </Step>
  <Step title="Invite a Member">
    Click **\+ Add Person**, enter their email address, and select their permission level.
  </Step>
</Steps>

<Tip>
  Set an hourly cost for users now if you want to track project profitability immediately.
</Tip>
