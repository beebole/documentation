Beebole's Kanban board lets you manage tasks visually by organizing them into columns based on their status. You can drag and drop task cards from one column to another as work progresses — for example, from **To Do** to **In Progress** to **Done**. The board gives your team a clear, real-time picture of where every task stands.

<Info>
  The Kanban board is one of three task views available in Beebole, alongside the [Gantt chart](/help/documentation/gantt) and the calendar view. All views share the same underlying task data, so changes in one view appear instantly in the others.
</Info>

---

## Accessing the Kanban board

<Steps>
  <Step title="Open a project">
    Click **Projects** in the left sidebar, then select the project whose tasks you want to manage.
  </Step>
  <Step title="Switch to the Kanban view">
    Click the **Kanban** tab at the top of the task area. The board displays your tasks organized by status columns.
  </Step>
</Steps>

---

## Understanding the board layout

The Kanban board is organized into vertical columns, one for each task status configured in your project. Tasks appear as cards within these columns.

- **Columns** — Each column represents a status in your workflow (for example, **To Do**, **In Progress**, **Review**, **Done**). The order of columns follows the status order you define in your project settings.
- **Task cards** — Each card shows the task name, the assigned person, and key details like the due date or priority. Click a card to open the full task detail panel.
- **Card count** — The number displayed at the top of each column indicates how many tasks are currently in that status.

## Configuring task statuses

Statuses define the columns on your Kanban board. You can customize the status workflow to match how your team actually works.

<Steps>
  <Step title="Open status settings">
    From the project's task area, click the **Settings** icon or go to the project settings. Navigate to the **Statuses** section.
  </Step>
  <Step title="Add or edit statuses">
    Click **Add status** to create a new column. Enter a **Name** for the status. Drag statuses to reorder them — the order here determines the column order on the board.
  </Step>
  <Step title="Remove a status">
    Click the delete icon next to a status you no longer need. You must first reassign any tasks in that status to another one.
  </Step>
</Steps>

<Tip>
  Keep your workflow simple. Three to five statuses (such as **To Do**, **In Progress**, **Review**, **Done**) work well for most teams. Adding too many columns can make the board harder to scan.
</Tip>

## Working with task cards

### Creating a task

<Steps>
  <Step title="Add a card to a column">
    Click the **+** button at the top or bottom of any status column. A new task card appears in that column.
  </Step>
  <Step title="Enter task details">
    Type a **Name** for the task. You can also set the **Assigned to** person, **Start date**, **End date**, and **Description** from the card detail panel.
  </Step>
  <Step title="Save the task">
    Click outside the card or press **Enter** to save. The task is now visible on the board and in all other task views.
  </Step>
</Steps>

### Moving a task between statuses

Drag a task card from one column and drop it into another to update its status. For example, drag a card from **To Do** to **In Progress** when work begins.

The status change takes effect immediately. Any filters, reports, or views that reference task status reflect the update in real time.

### Editing a task

Click any task card to open its detail panel. From here you can update:

- **Name** and **Description** — Edit the task title or add a rich text description with formatting.
- **Assigned to** — Change the person responsible for the task.
- **Start date** and **End date** — Adjust the task's scheduled timeline.
- **Status** — Change the status using a dropdown (an alternative to drag and drop).
- **Custom fields** — Fill in any task-level custom fields configured for the project. See [Custom fields](/help/documentation/custom-fields).

## Task descriptions

Every task has a **Description** field that supports full rich text editing. Click the **Description** area in the task detail panel to open the editor.

The editor is powered by Lexical and supports:

- Text formatting — bold, italic, underline, strikethrough
- Headings (H1, H2, H3)
- Ordered and unordered lists
- Code blocks with syntax highlighting
- Links
- Inline images, videos, and PDF attachments (drag and drop or paste)
- @mentions to reference team members

The description is shared across all task views — edits you make on the Kanban board are immediately visible when the same task is opened from the Gantt chart or the planning view.

<Tip>
  Use descriptions to capture acceptance criteria, reference links, or background context. The description panel is separate from comments, so it works well for stable task-level information.
</Tip>

## Task-level custom fields

Custom fields let you extend tasks with additional structured data beyond the built-in fields. A custom field is defined once in your account settings and then made visible on specific task categories.

### Supported field types

| Type | What it stores |
|------|---------------|
| **Text** | Free-form text, with optional length constraints and allowed value lists |
| **Number** | Numeric value with optional validation rules |
| **Date** | A date picker value |
| **Date & time** | A date and time picker value |
| **URL** | A validated URL |
| **Yes / No** | A boolean toggle |

### Configuring custom fields for tasks

Custom fields are configured in **Settings > Custom fields**. When creating or editing a field, set its visibility to include the task categories where it should appear. Once visible, the field appears in the task detail panel for every task in those categories.

### Filling in custom field values

Open a task detail panel. Scroll to the custom fields section. Click any field to enter or select a value. Values are saved immediately when you leave the field.

<Info>
  Custom fields on tasks are separate from custom fields on projects or people. Each entity type has its own visibility configuration.
</Info>

## Setting WIP limits

WIP (Work In Progress) limits let you cap the number of tasks allowed in a status column at any time. This prevents bottlenecks and encourages your team to finish tasks before starting new ones.

<Steps>
  <Step title="Open the column settings">
    Click the column header or the settings icon on the status column where you want to set a limit.
  </Step>
  <Step title="Set the WIP limit">
    Enter a maximum number of tasks for this column. For example, setting a WIP limit of 5 on the **In Progress** column means no more than 5 tasks should be in progress at once.
  </Step>
</Steps>

When a column exceeds its WIP limit, Beebole highlights the column header as a visual warning. You can still add tasks beyond the limit, but the warning signals that the team may be overloaded.

<Info>
  WIP limits are guidelines, not hard blocks. Beebole highlights the column but does not prevent you from moving cards into an over-limit column. This keeps your workflow flexible while encouraging discipline.
</Info>

## Task ownership and assignment

Every task can be assigned to one person. The assigned person's name and avatar appear on the task card, making it easy to see who is responsible for what at a glance.

To filter the board by person, use the **Filter by person** option above the board. This is helpful when reviewing workload for a specific team member.

---

## Bulk operations

You can perform actions on multiple tasks at once to save time:

- **Select multiple cards** — Use the checkbox on each card or use **Select all** within a column.
- **Change status** — Move all selected tasks to a different status column.
- **Reassign** — Change the assigned person for all selected tasks.
- **Delete** — Remove all selected tasks.

<Warning>
  Deleting tasks is permanent. Deleted tasks are removed from all views, including the [Gantt chart](/help/documentation/gantt) and calendar.
</Warning>

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Can I use the Kanban board without setting up statuses first?">
    Beebole provides a default set of statuses (such as **To Do**, **In Progress**, and **Done**) when you create a project. You can start using the board immediately and customize statuses later.
  </Accordion>
  <Accordion title="Do changes on the Kanban board affect the Gantt chart?">
    Yes. The Kanban board and [Gantt chart](/help/documentation/gantt) share the same task data. Changing a task's status, dates, or assignment on the board updates it everywhere.
  </Accordion>
  <Accordion title="Can I limit who can move tasks on the board?">
    Yes. Use [Roles & permissions](/help/documentation/roles-authorisations) to control who can edit tasks. People with read-only access can view the board but cannot move cards.
  </Accordion>
  <Accordion title="What happens when I archive a project that has tasks on the Kanban board?">
    Archiving a project hides it and all its tasks from active views, including the Kanban board. The data is preserved and reappears if you unarchive the project.
  </Accordion>
  <Accordion title="Is there a limit to the number of tasks on the Kanban board?">
    There is no hard limit. For large projects, use filters to narrow the view by person, status, or date range.
  </Accordion>
  <Accordion title="How do I add a custom field to tasks?">
    Go to **Settings > Custom fields**, create or edit a field, and set its visibility to include the relevant task categories. The field then appears in the task detail panel for all tasks in those categories.
  </Accordion>
  <Accordion title="Can multiple team members edit a task description at the same time?">
    The description editor is single-user at a time. If two people open the same task simultaneously, the last save wins. For collaborative notes, use the task's comment or journal feed instead.
  </Accordion>
</AccordionGroup>
