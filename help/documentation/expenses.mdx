Beebole's expense tracking lets you record and manage project-related expenses alongside time entries. Expenses are categorized by type and can be included in reports for a complete picture of project costs.

---

## Expense types

Before your team can log expenses, you need to define the expense types they'll use. Expense types categorize spending into meaningful groups.

### Creating expense types

<Steps>
  <Step title="Open Settings">
    Go to **Settings** > **Expense types**.
  </Step>
  <Step title="Add an expense type">
    Click **[+]** and enter a name for the type (e.g., Travel, Meals, Equipment, Software licenses).
  </Step>
  <Step title="Configure the type">
    Choose whether the expense is **monetary** (a currency amount) or **quantity-based** (units, like kilometers or meals). Optionally set a **markup percentage** for client billing.
  </Step>
  <Step title="Save">
    Click **Save**. The expense type is now available for logging.
  </Step>
</Steps>

---

## Controlling expense types per project

Not every expense type applies to every project. You can control which expense types are available on each project through [Assignments](/help/documentation/assignments):

- By default, all expense types are available on all projects.
- Restrict specific types to specific projects when needed.
- Override availability at the tag or person level.

## Logging expenses

Team members log expenses from their timesheet or from the project view:

<Steps>
  <Step title="Select the project">
    Choose the project the expense relates to.
  </Step>
  <Step title="Add an expense entry">
    Select the expense type, enter the amount (or quantity), date, and an optional description.
  </Step>
  <Step title="Save">
    The expense is recorded and included in project cost tracking.
  </Step>
</Steps>

## Expenses in reports

Expense data appears in [Reports](/help/documentation/reports) when you select the **Expense records** dimension. You can group and filter expenses by project, person, expense type, and date range.

<Info>
  Expenses with markup are automatically calculated in reports — showing both the base cost and the marked-up amount for client billing.
</Info>

---

## Budget tracking for expenses

Expense amounts contribute to project [budgets](/help/documentation/budgets) when cost budgets are configured. This means you can monitor total project spending — including both labor costs and expenses — against a single budget target.

To track expenses against a budget, set a **cost budget** on the project. As team members log expenses, the amounts are added to the budget consumption alongside time-based cost calculations.

<Tip>
  Review budget progress regularly in [Reports](/help/documentation/reports) to catch overspending early. Beebole does not block expense entries when a budget is exceeded.
</Tip>

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Can expenses go through the approval workflow?">
    Yes. Expenses are included in the [approval workflow](/help/documentation/approval) alongside time entries.
  </Accordion>
  <Accordion title="Can I set a budget for expenses?">
    Expense amounts contribute to project [budgets](/help/documentation/budgets) when cost budgets are configured.
  </Accordion>
  <Accordion title="Can I restrict who can log certain expense types?">
    Yes. Use [Assignments](/help/documentation/assignments) to control expense type availability by person, tag, or project.
  </Accordion>
  <Accordion title="How does markup work on expenses?">
    When you configure a markup percentage on an expense type, Beebole automatically calculates the marked-up amount for client billing. Reports show both the base cost and the marked-up amount.
  </Accordion>
</AccordionGroup>
